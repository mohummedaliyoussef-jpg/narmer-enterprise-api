# 🪓 استراتيجية هندسة الفوضى (Chaos Engineering)

## المفهوم
ممارسة تعمّد إحداث أعطال في النظام لاختبار صموده وقدرته على التعافي، مما يضمن موثوقية عالية في الإنتاج.

## الأداة المقترحة: Chaos Mesh (على Kubernetes)
- **التنصيب:** `helm install chaos-mesh chaos-mesh/chaos-mesh`
- **التجارب الجاهزة:** فصل الشبكة، إنهاء pods، تأخير الإدخال/الإخراج.

## سيناريوهات الاختبار المقترحة
| السيناريو | الهدف |
|-----------|-------|
| قتل Redis فجأة | التأكد من أن التطبيق يعمل بشكل متدهور (Degraded) بدلاً من الانهيار الكامل. |
| فصل PostgreSQL | التحقق من وضوح رسائل الخطأ وإعادة الاتصال التلقائي. |
| زيادة تحميل API | اختبار فعالية التوسع الأفقي التلقائي (HPA). |
| فصل Elasticsearch | التأكد من تسجيل الأخطاء وعدم توقف التطبيق. |

## التقديم للمدقق
> "بعد النشر على Kubernetes، سيتم تطبيق **هندسة الفوضى** باستخدام Chaos Mesh لضمان صمود النظام تحت أي ظرف، وفقًا للسيناريوهات الموثقة في خطة الاختبار."
